# Кейс 2. RPA-робот для создания документов в 1С

Робот автоматически создаёт документы «Счёт покупателю» в 1С на основе
входных данных из JSON-файла. Для каждого счёта робот через веб-интерфейс
1С заполняет шапку и табличную часть, ведёт отчёт и журнал логов.

## Что делает робот

1. Читает список счетов из файла `orders.json`.
2. Заходит (авторизуется) в 1С через браузер.
3. Открывает список «Счета покупателям».
4. По каждому счёту: создаёт новый документ, заполняет контрагента,
   организацию, договор и товары.
5. Записывает результат в CSV-отчёт и журнал логов.
6. При ошибке на одном документе не останавливается — фиксирует ошибку
   и продолжает со следующего.

## Структура входного файла (orders.json)

```json
[
  {
    "number": "",
    "date": "2025-04-01T00:00:00",
    "organization": "Торговый дом \"Комплексный\" ООО",
    "counterparty": "Турбаза Яхрома",
    "contract": "20 от 09.01.2024",
    "items": [
      {"product": "Печенье шоколадное", "quantity": 150, "price": 180},
      {"product": "Конфеты Батончик", "quantity": 26, "price": 150}
    ]
  }
]
```

- `number` — номер счёта (можно оставить пустым, 1С присвоит сама)
- `date` — дата документа
- `organization` — организация
- `counterparty` — контрагент (должен существовать в базе)
- `contract` — договор
- `items` — список товаров (наименование, количество, цена)

## Как запустить

1. Откройте проект в Puzzle RPA Studio (версия 3.0.3).
2. Положите файл `orders.json` в папку `resource/`.
3. Укажите данные доступа к базе 1С в блоках авторизации (логин, пароль,
   адрес базы).
4. Запустите процесс `main` кнопкой «Запустить».
5. Робот откроет браузер, войдёт в 1С и начнёт создавать документы.

## Результаты работы

- `resource/result.csv` — отчёт: номер документа, статус (Успех/Ошибка),
  сообщение.
- Папка `logs/` — журнал работы (INFO — успех, ERROR — ошибки) с итоговой
  сводкой: сколько документов создано, сколько с ошибкой.

## Архитектура

Робот построен по принципу разделения этапов:
- **Чтение данных** — отдельный блок чтения JSON.
- **Авторизация** — выполняется один раз перед обработкой.
- **Создание документов** — цикл по списку, внутри — заполнение полей.
- **Запись результата** — отчёт и логи.

Взаимодействие с интерфейсом 1С реализовано через веб-блоки с поиском
элементов по XPath. Тело документа собирается из переменных стандартными
блоками, без Python-скриптов, что упрощает поддержку и расширение.

## Обработка ошибок

Каждый документ обрабатывается в блоке перехвата ошибок. Если создание
не удалось (нет контрагента, ошибка валидации и т.п.), ошибка
записывается в отчёт и лог, и робот переходит к следующему документу —
сбой одного счёта не прерывает всю работу.

## Особенности реализации

Часть кнопок интерфейса (создание документа, добавление строки)
обрабатывается через горячие клавиши и XPath по смысловым идентификаторам
элементов формы.

## Технологии

- Puzzle RPA Studio 3.0.3
- Веб-автоматизация (браузер Chrome)
- Формат входных данных: JSON
- Формат отчёта: CSV
