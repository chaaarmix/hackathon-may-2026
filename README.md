# Расширение: Форматировать Word по шаблону

## Описание
Блок автоматически приводит Word-документы к единому стилю: 
очищает лишнее форматирование, применяет заданный шрифт, 
определяет заголовки и форматирует таблицы. 
Поддерживает замену плейсхолдеров вида {{ключ}} на реальные значения.
Предназначен для автоматизации подготовки документов в процессах Puzzle RPA.

## Требования к окружению
- Операционная система: Windows 10/11
- Установлен Microsoft Word (2016 или новее)
- Puzzle RPA Studio 3.0.2

## Установка расширения
1. Скачайте архив `WordFormatterExtension.zip` и распакуйте его в любую папку.
2. Откройте Puzzle RPA Studio → **Настройки** → **Расширения** → **Добавление**.
3. Нажмите **Выбрать** и укажите папку с распакованным расширением.
4. Выберите категорию «Интеграция приложений/Word».
5. Перезапустите Puzzle RPA Studio.

После перезапуска блок появится в тулбоксе в категории «Интеграция приложений → Word».

## Параметры блока

| Параметр | Тип | Обязательный | Значение по умолчанию | Описание |
|----------|-----|--------------|----------------------|----------|
| input_path | string | Да | — | Полный путь к исходному `.docx` файлу |
| output_path | string | Нет | — | Путь для сохранения результата (если не указан, исходный файл перезаписывается) |
| font_name | string | Нет | Calibri | Основной шрифт документа |
| font_size | int | Нет | 12 | Размер шрифта в пунктах |
| replacements | dict | Нет | — | Словарь для замены плейсхолдеров, формат: `{"ключ": "значение"}` |

## Пример использования в Puzzle RPA

1. Перетащите блок на рабочую область.
2. Заполните обязательные поля:
   - `input_path`: `C:\test\my_document.docx`
   - (опционально) `output_path`: `C:\test\result.docx`
3. При необходимости укажите `font_name` и `font_size`.
4. Если в документе есть плейсхолдеры, например `{{Дата}}`, передайте в `replacements`:
{"Дата": "20.05.2025"}
5. Запустите выполнение (F5).

## Что делает блок
1. Очищает документ: удаляет пустые абзацы, табуляции, лишние пробелы, сбрасывает прямое форматирование.
2. Применяет единый шрифт ко всему тексту.
3. Автоматически определяет заголовки по нумерации (`1.`, `1.1.`) и ключевым словам (`ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ` и др.) и присваивает им стили «Заголовок 1» и «Заголовок 2».
4. Форматирует таблицы: включает границы, выравнивает содержимое, выделяет заголовочную строку жирным шрифтом и заливкой.
5. Заменяет плейсхолдеры вида `{{ключ}}` на значения из словаря.

## Демонстрационные файлы
- `test_input.docx` — тестовый документ до обработки.
- `test_output.docx` — результат работы блока на этом документе (эталон).

## Известные ограничения
- Требуется установленный Microsoft Word 2016 или новее.
- Работает только на Windows.
- Гарантированно поддерживается формат `.docx`.
- Файлы других форматов (`.doc`, `.dot`, `.dotx`, `.docm`, `.dotm`) могут быть открыты, но корректность сохранения не гарантируется.
- Если Word завис в фоне, блок автоматически перезапускает его.
- Если `output_path` не указан, исходный файл перезаписывается.
